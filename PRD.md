# PRD: "AI恋爱军师" H5应用需求文档

## 1. 项目背景与目标

### 1.1 项目简介
"AI恋爱军师"是一款H5网页应用，旨在通过分析目标对象的社交媒体（如朋友圈）截图，利用大语言模型提供深度见解和行动建议，从而帮助用户更好地了解对方，并成功建立亲密关系。

### 1.2 项目目标
核心目标是利用AI技术，为用户在建立恋爱关系的过程中提供策略支持，提高社交成功率。具体来说，就是帮助用户与他们感兴趣的人从朋友发展成为男女朋友。

### 1.3 目标用户
希望通过分析社交媒体内容来更深入了解某人，并寻求与之建立恋爱关系的单身人士。

## 2. 功能需求 (Functional Requirements)

### 2.1 图片上传功能
- **基本要求**: 用户可以在网页上选择并上传图片。
- **多图上传**: 支持一次性选择多张图片。
- **数量限制**: 必须上传至少5张图片才能进行分析，UI界面需有明确提示和校验。
- **预览与管理**: 上传后，页面应显示图片的缩略图，并允许用户删除或替换已上传的图片。

### 2.2 AI分析功能
- **模型调用**:
    - 前端将用户上传的图片（建议进行Base64编码）发送至后端服务器。
    - 后端服务器接收到图片数据后，调用火山方舟的豆包大模型 (`doubao-seed-1.6-thinking`)。
    - **注意**: API Key等敏感信息必须存储在后端，严禁在前端代码中暴露。
- **Prompt设计**:
    - 发送给大模型的指令(Prompt)需要精心设计，以引导模型生成有针对性的、以建立恋爱关系为目的的分析报告。例如：
      > "你是一位资深的恋爱顾问。请基于以下这些朋友圈截图，深入分析这个人的性格特点、兴趣爱好、价值观，并提供一些实用的建议，帮助我与TA发展成为男女朋友。建议可以包括：如何开启对话、可以聊哪些话题、适合的约会活动、以及追求过程中需要注意什么。"
- **API接口**:
    - **模型名称**: `doubao-seed-1.6-thinking`
    - **调用文档**: `https://www.volcengine.com/docs/82379/1399008`

### 2.3 分析报告展示
- **加载状态**: 在等待模型返回结果期间，界面应有明确的加载动画或提示，告知用户“正在分析中...”。
- **结构化展示**: 分析结果应该以清晰、易于阅读的卡片或段落形式呈现。建议包含以下几个模块：
    - **性格画像**: 总结此人的性格特征（如：外向、热爱生活、文艺、务实等）。
    - **兴趣图谱**: 罗列出TA的兴趣爱好（如：旅行、美食、健身、音乐、阅读等）。
    - **破冰话题建议**: 提供几个可以用来开启对话或深入交流的话题。
    - **约会活动推荐**: 根据TA的兴趣，推荐一些可能成功的约会方案。
    - **恋爱攻略**: 综合分析后，提供一些追求对方的策略性建议和注意事项。

## 3. 非功能需求 (Non-Functional Requirements)

### 3.1 隐私与安全
- **隐私声明**: 应用必须有清晰的隐私政策，告知用户上传的图片将被用于AI分析。
- **数据处理**: 明确告知用户，图片仅用于当次分析，服务器不会永久保存用户的上传内容和分析结果。
- **密钥安全**: 调用大模型的API Key必须安全地存储在后端服务器，防止泄露。

### 3.2 性能
- **响应速度**: 图片上传和处理应尽可能高效。
- **用户体验**: 由于大模型分析可能需要一定时间（5-15秒），前端必须做好异步处理和加载状态管理，避免页面假死。

### 3.3 界面与交互 (UI/UX)
- **设计风格**: 界面应简洁、美观，符合移动端（H5）用户的使用习惯。
- **引导清晰**: 操作流程应有清晰的文字引导，让用户轻松上手。

## 4. 技术栈设想

- **前端**: HTML, CSS, JavaScript。可考虑使用轻量级框架（如Vue.js）以提升开发效率。
- **后端**: **必须有后端服务**。可使用 Node.js (Express/Koa) 或 Python (Flask/FastAPI) 等技术，用于安全地调用外部大模型API。
- **API**: 火山方舟豆包大模型 API。
